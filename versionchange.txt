# versionchange.txt
#
# Checklist of files that carry the project version string and must be bumped
# on every version change. Tracked in-repo as a maintenance aid.
#
# Runtime version strings (ATI / ATI3, Kermit identity, webserver banner, GUI
# title, startup log) all derive from env!("CARGO_PKG_VERSION"), so bumping
# Cargo.toml updates them automatically — the doc files below must be matched
# by hand.
#
# Current version: 0.6.3
#
# ── Source of truth ─────────────────────────────────────────────
Cargo.toml              # package version (line 3 only — NOT the russh/dep "0.60" specs); Cargo.lock follows on build
versionchange.txt       # the "Current version:" line in this file's header
#
# ── Documentation (match Cargo.toml by hand) ────────────────────
README.md               # download/verify example artifact filenames (ethernet-gateway-vX.Y.Z-...)
usermanual.html         # .version header, sha256 example, Kermit identity, ATI/ATI3 strings, footer
#                         DO NOT bump the historical refs: "removed in v0.6.0", "prior to v0.6.0"
usermanual.pdf          # regenerate from the updated HTML — same tool as before:
#                         weasyprint usermanual.html usermanual.pdf   (Producer must stay "WeasyPrint")
web/index.html          # <title>, .hdr-version, startup-banner output-text, footer
web/appendix.html       # .hdr-version, footer
web/kermit.html         # <title>, .hdr-version, "Ethernet Gateway Kermit X.Y.Z" identity strings, footer
web/kermitreference.html
web/xmodemreference.html
web/ymodemreference.html
web/zmodemreference.html

web/telnetreference.html

web/sshreference.html        # <title>, .hdr-version, footer
#
# ── On release (separate step) ──────────────────────────────────
CHANGELOG.md            # Pre-release, the next version's notes accumulate under a
#                         "## [X.Y.Z] - Unreleased" heading. At release time:
#                           1. Change "- Unreleased" to the release date: "## [X.Y.Z] - YYYY-MM-DD"
#                           2. Repoint the link ref from compare/...HEAD to the tag:
#                              [X.Y.Z]: https://github.com/rickybryce/ethernet-gateway/releases/tag/vX.Y.Z
#                           3. Keep the [Unreleased] line for the next cycle's work
#                         Then tag vX.Y.Z and merge dev -> master to trigger the Release workflow.
#
# Verify after a bump:
#   grep -rn '0\.6\.1' web/ README.md   # expect: none
#   grep -n  '0\.6\.0' usermanual.html  # expect: ONLY the two historical refs
#   cargo build                          # confirms Cargo.toml parsed + new version compiled
